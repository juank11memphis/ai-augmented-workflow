# Step: Create the maintainer release document and outline

## Goal
Create one dedicated maintainer-facing release document so npm publication steps stop living in tribal knowledge or scattered notes.

## Scope
- Choose a single maintainer doc location that keeps `README.md` end-user focused.
- Create the document with a clear title, purpose, and section outline.
- Introduce the release doc as the maintainer-facing source for npm publication steps.
- Do not fully write the detailed release flow in this step.

## Files
- maintainer release document file (recommended: `docs/releasing.md`)
- `README.md` only if a brief pointer is needed

## Done when
- The repo contains one obvious maintainer release doc.
- The document clearly identifies itself as maintainer-only release guidance.
- The chosen location keeps end-user install/update guidance separate from release operations.
